Introduction

In this tutorial I will show you how to create a basic mod in Minecraft Forge.

This tutorial was made on 12/05/2016 (DD/MM/YYYY) using Minecraft Forge 1.9-12.16.1.1901 although this tutorial envolves a lot of things that haven't changed for a long time and probably won't any time soon. If something does change in the future I will update the tutorial.

All the code for these tutorials will be located on my Github repository on separate branches for each tutorial.

Things you will need

A project that has been setup with Forge and either Eclipse or IntelliJ IDEA, if you don't have this you can learn how to do this on the Getting Started page

Creating the main mod file

The main mod file is the file that Forge will use to load your mod.

This file is usually named by the name of your mod or project name. So I will be naming it MinecraftForgeTutorials

Before we create this file we need to create a package for the mod. This will be located in "/src/main/java/", navigate to this folder in your IDE.

You should decide on a name for your package. The naming convention for packages is your domain name reversed then your project name so for example I would use "uk.co.wehavecookies56.minecraftforgetutorials"

In Eclipse right click the folder and go to "New > Package", type your package name and click "Finish"

In IDEA right click the folder and go to "New > Package" type your package name and click "OK"

Now we can create the main mod file.

In Eclipse right click the package and go to "New > Class", type your class name and click "Finish"

In IDEA right click the package and got to "New > Java Class", type your class name and click "OK"

It will ask if you want to add the file to Git this will stage the file for you so you don't have to use the "git add" command for the file.

Now that the class has been created we need to add the Mod annotation to the class. With the elements "name", "modid" and "version". Here's what it should look like.

          @Mod(name = "Minecraft Forge Tutorials", modid = "mft", version = "1.0")
          public class MinecraftForgeTutorials {

Your "modid" is a unique identifier for your mod that shouldn't contain any special characters or upper case characters. I used "mft" as it is short and easy to type. The other two don't matter too much although your version number should probably follow some form of version conventions.

You will also need to import net.minecraftforge.fml.common.Mod which you can do in Eclipse with either Ctrl+Shift+O or hovering over the @Mod and click "import net.minecraftforge.fml.common.Mod". In IDEA press Alt+Enter and click "Import class". Of course you can also manually type in the imports.

Your mod file will need to contain three main methods which are called during the three stages of the initialization of the game. These are the preInit, init and postInit.

preInit is short for Pre Initialisation, init is short for Initialisation and postInit is short for Post Initialisation. The pre initialisation and the initialisation are called during the loading of the game. The post initialisation is called after the game has loaded.

These methods need the EventHandler annotation as well as a parameter with the type (Pre/Post)InitilizationEvent. They do not need a return type and will need to be imported as well.

Here's what the methods should look like.

          @Mod.EventHandler
          public void preInit(FMLPreInitializationEvent event) {

          }

          @Mod.EventHandler
          public void init(FMLInitializationEvent event) {

          }

          @Mod.EventHandler
          public void postInit(FMLPostInitializationEvent event) {

          }

That's all you need for the main mod file for now.

This is what the final class should look like.

          package uk.co.wehavecookies56.minecraftforgetutorials;

          import net.minecraftforge.fml.common.Mod;
          import net.minecraftforge.fml.common.event.FMLInitializationEvent;
          import net.minecraftforge.fml.common.event.FMLPostInitializationEvent;
          import net.minecraftforge.fml.common.event.FMLPreInitializationEvent;

          @Mod(name = "Minecraft Forge Tutorials", modid = "mft", version = "1.0")
          public class MinecraftForgeTutorials {

              @Mod.EventHandler
              public void preInit(FMLPreInitializationEvent event) {

              }

              @Mod.EventHandler
              public void init(FMLInitializationEvent event) {

              }

              @Mod.EventHandler
              public void postInit(FMLPostInitializationEvent event) {

              }
          }

Creating the mcmod.info

The mcmod.info is a file that is used to provide information about your mod on the mod GUI located on the main menu.

Create a new file called "mcmod.info" in "src/main/resources/" make sure the file has the ".info" file extension.

This is an example of what the file should contain

          [
          {
            "modid": "mft",
            "name": "Minecraft Forge Tutorials",
            "description": "The mod made for my Minecraft Forge Tutorials",
            "version": "${version}",
            "mcversion": "${mcversion}",
            "url": "https://github.com/Wehavecookies56/Minecraft-Forge-Tutorials",
            "updateUrl": "",
            "authorList": ["Wehavecookies56"],
            "credits": "Credit to the Forge and FML guys for being awesome.",
            "logoFile": "",
            "screenshots": [],
            "requiredMods": [],
            "dependencies": [],
            "dependants": [],
            "useDependencyInformation": true
          }
          ]

"modid" and "name" should be the same as what you set it as in the Mod annotation. "version" and "mcversion" is the version of the mod and the version of Minecraft the mod is for, these are set as "${version}" and "${mcversion}" which will be automatically changed to the version and mcversion from the build.gradle when your mod is built. The other elements are fairly simple to fill out. "requiredMods", "dependencies" and "dependants" should use the mod id of the mod. The elements which are arrays should have commas separating each value for example if you have more than one mod author "authorList" would have "["Wehavecookies56", "Author2", "Author3"]"

That's it for the mcmod.info file.

Creating the proxies

Proxies are used for initialising things on a specifically on either the server or client. The Client Proxy is the most helpful one as it will be used for anything to do with registering textures, models, etc.

We will be creating a new package inside our current package named "proxies". Then two files named "CommonProxy" and "ClientProxy". The ClientProxy class should extend CommonProxy. The CommonProxy should also contain a preInit, init and postInit methods. They should take the same arguments as the methods in the main mod file. These methods should be overriden in the ClientProxy which can be done quickly in IDEA by pressing Ctrl+O and selecting the methods you want to override, the super statements are not needed. These two classes should like these:

CommonProxy

          package uk.co.wehavecookies56.minecraftforgetutorials.proxies;

          import net.minecraftforge.fml.common.event.FMLInitializationEvent;
          import net.minecraftforge.fml.common.event.FMLPostInitializationEvent;
          import net.minecraftforge.fml.common.event.FMLPreInitializationEvent;

          public class CommonProxy {

              public void preInit(FMLPreInitializationEvent event) {

              }

              public void init(FMLInitializationEvent event) {

              }

              public void postInit(FMLPostInitializationEvent event) {

              }
          }

ClientProxy

          package uk.co.wehavecookies56.minecraftforgetutorials.proxies;

          import net.minecraftforge.fml.common.event.FMLInitializationEvent;
          import net.minecraftforge.fml.common.event.FMLPostInitializationEvent;
          import net.minecraftforge.fml.common.event.FMLPreInitializationEvent;

          public class ClientProxy extends CommonProxy {

              @Override
              public void preInit(FMLPreInitializationEvent event) {
                  
              }

              @Override
              public void init(FMLInitializationEvent event) {
                  
              }

              @Override
              public void postInit(FMLPostInitializationEvent event) {
                  
              }
          }

Now we have these classes created, we need to call these three methods as well as register our proxies in our main mod file.

The proxy is registered using the SidedProxy annotation like so

          @SidedProxy(clientSide = "uk.co.wehavecookies56.minecraftforgetutorials.proxies.ClientProxy", serverSide = "uk.co.wehavecookies56.minecraftforgetutorials.proxies.CommonProxy")
          public static CommonProxy proxy;

This should be inside your main mod class outside all of the methods. The "clientSide" and "serverSide" elements are set to the locations of each of the proxies.

"clientSide" should be the location of your ClientProxy and which is the "package name" + ".ClientProxy" and "serverProxy" should be the same but with the Common Proxy instead.

We then need to call the methods from the proxies in the initialisation methods.

          proxy.preInit(event);

And the same for each one.

That it for the proxies. The final main mod class should look like this

          package uk.co.wehavecookies56.minecraftforgetutorials;

          import net.minecraftforge.fml.common.Mod;
          import net.minecraftforge.fml.common.SidedProxy;
          import net.minecraftforge.fml.common.event.FMLInitializationEvent;
          import net.minecraftforge.fml.common.event.FMLPostInitializationEvent;
          import net.minecraftforge.fml.common.event.FMLPreInitializationEvent;
          import uk.co.wehavecookies56.minecraftforgetutorials.proxies.CommonProxy;

          @Mod(name = "Minecraft Forge Tutorials", modid = "mft", version = "1.0")
          public class MinecraftForgeTutorials {

              @SidedProxy(clientSide = "uk.co.wehavecookies56.minecraftforgetutorials.proxies.ClientProxy", serverSide = "uk.co.wehavecookies56.minecraftforgetutorials.proxies.CommonProxy")
              public static CommonProxy proxy;

              @Mod.EventHandler
              public void preInit(FMLPreInitializationEvent event) {
                  proxy.preInit(event);
              }

              @Mod.EventHandler
              public void init(FMLInitializationEvent event) {
                  proxy.init(event);
              }

              @Mod.EventHandler
              public void postInit(FMLPostInitializationEvent event) {
                  proxy.postInit(event);
              }
          }

Summary

Currently our mod doesn't do anything other than load which isn't very interesting, but now that we have the basics complete we can get on to the interesting stuff.
